\chapter{研究成果}

本章从暗物质密度场和速度场两个方面评估本文快速模拟流程的表现。由于维里化的存在，最小尺度的结构不能被模拟逐点准确预测，N 体模拟也并非在逐点意义上完全可复现，故本文主要关注小尺度结构的统计特征。密度场部分依次考察切片视觉效果、过密度残差概率密度函数、功率谱、双谱以及功率谱协方差矩阵；速度场部分考察速度模切片以及速度散度、旋度、速度模和速度分量的功率谱。除特别说明外，统计量均以 Quijote 低精度 N 体模拟为参考\citep{villaescusa2020quijote}，并将新流程与 ZA、2LPT 和 ALPT 等微扰论基准进行比较。

\section{暗物质场单点统计}

首先比较过密度场
\(\delta(\boldsymbol{x})=\rho(\boldsymbol{x})/\bar{\rho}-1\)
的单点统计和切片形态。其中 \(\rho(\boldsymbol{x})\) 是位置 \(\boldsymbol{x}\) 处的暗物质密度，\(\bar{\rho}\) 是全模拟盒子的平均密度。图~\ref{fig:density-slice} 展示了一个模拟实现的过密度切片。N 体模拟中可以看到清晰的丝状结构、高密度团块和空洞区域；ZA 基准能够保留大尺度结构的位置，但小尺度团块明显更弥散，高密度峰值也更平滑。新流程在整体结构位置上与 ZA 保持一致，同时显著增强了丝状结构和团块的紧致程度，在视觉上更接近 N 体模拟。

\section{暗物质场两点统计}

两点统计用物质功率谱 \(P(k)\) 表征，定义为
\begin{equation}
    P(k) = \langle |\delta(\boldsymbol{k})|^2 \rangle,
\end{equation}
其中 \(\delta(\boldsymbol{k})\) 是过密度场的傅里叶变换，\(\boldsymbol{k}\) 是波矢，\(k=|\boldsymbol{k}|\) 是波数；\(\langle\cdot\rangle\) 表示对同一 \(k\) 壳层内所有傅里叶模式取平均。由于模拟场的离散化，功率谱在高波数端会受到采样噪声影响，由Nyquist频率限制。在本文环境中，Nyquist频率对应波数为 \(k_{\mathrm{Nyquist}} = \pi N_{\mathrm{grid}}/L=0.804\,h\,\mathrm{Mpc}^{-1}\)，其中 \(N_{\mathrm{grid}}=256\) 是网格数，\(L=1\,h^{-1}\mathrm{Gpc}\) 是模拟盒子边长，\(k_{\mathrm{Nyquist}}\)之上的波数无法可靠估计。图~\ref{fig:density-power} 给出了共16个 模拟实现的功率谱比值 \(P/P_{\mathrm{ref}}\)。在大尺度低波数区域，各方法都接近 N 体参考结果，这与微扰论在弱非线性尺度上有效的预期一致。随着波数增大，ZA、2LPT 和 ALPT 的功率谱比值逐渐低于 1，说明这些方法在小尺度上系统性低估密度涨落功率。

相比之下，新流程在图示波数范围内始终保持在 1 附近，仅在最高波数端出现轻微下降。这说明残差网络有效补偿了 ZA 基准缺失的小尺度非线性功率，并且没有明显破坏大尺度模式。结合图~\ref{fig:density-slice} 的视觉结果可以看出，功率谱上的改进主要来自高密度团块和细丝结构的恢复。

虽然 TUNeS 方法使用了 2 倍于当前的粒子分辨率，但其在红移 0 的 \(k\lesssim 0.8\,h\,\mathrm{Mpc}^{-1}\) 区间有时会超过 6\% 的相对误差\citep{kang2026tunes}；本文方法在红移 0 的表现始终处于 4\% 以内的误差水平。这表明本文架构在小尺度结构恢复方面具有更稳定和更高的准确性，尤其是在非线性演化较强的区域。

\section{暗物质场三点统计}

三点统计进一步检验非高斯结构的恢复情况。物质双谱 \(B(k_1,k_2,k_3)\) 定义为
\begin{equation}
    B(k_1, k_2, k_3) = \langle \delta(\boldsymbol{k}_1) \delta(\boldsymbol{k}_2) \delta(\boldsymbol{k}_3) \rangle,
\end{equation}
其中三个波矢满足构成矢量三角形的条件 \(\boldsymbol{k}_1 + \boldsymbol{k}_2 + \boldsymbol{k}_3 = 0\)，\(k_1, k_2, k_3\) 分别是三个波矢的模；\(\langle\cdot\rangle\) 表示对满足该三角形构型的傅里叶模式组合取平均。在比较时，我们固定其中两个波数 \(k_1\) 和 \(k_2\)，并将两波矢夹角 \(\theta\) 作为横轴。

图~\ref{fig:density-bispectrum} 比较了六种三角形构型下的双谱比值 \(B/B_{\mathrm{ref}}\)。在所有构型中，ZA、2LPT 和 ALPT 的比值普遍低于 1，且偏差随 \(k_1\) 和 \(k_2\) 增大而加重，说明微扰论方法不仅低估小尺度两点功率，也未能充分恢复非线性结构引入的三点关联。特别是在 \(k_1=k_2=0.40\,h\,\mathrm{Mpc}^{-1}\) 等更小尺度构型中，微扰论基准的双谱只达到 N 体参考的一小部分。

新流程在六个构型中均显著接近比值 1，尤其在低、中等波数构型下基本保持在 N 体参考附近。在较高波数构型中，新流程仍有轻微低估，但偏差远小于 ZA、2LPT 和 ALPT。这表明残差网络不仅恢复了功率谱幅度，也较好地恢复了由非线性引力演化带来的高阶相位耦合。

\section{暗物质功率谱协方差矩阵}

除均值统计外，协方差矩阵也是宇宙学参数推断中必须准确估计的量。这里使用共240个模拟实现估计暗物质功率谱协方差矩阵，并比较新流程与 N 体参考结果的差异。图~\ref{fig:density-covariance} 左上和右上分别给出 N 体参考与新流程的 \(\log_{10}|\mathrm{Cov}(P(k))|\)。两者在整体矩阵形态、低波数与高波数之间的相关结构以及对角线主结构上较为相似，说明新流程能够较好地保留不同尺度功率谱涨落之间的相关性。

更细致地看，图~\ref{fig:density-covariance} 左下显示新流程相对 N 体参考的协方差逐元差异在较大 \(k\) 区域整体偏负；右下的对角元比值也显示，新流程在小尺度上的方差低于 N 体参考。尽管如此，新流程的协方差对角元仍明显高于 ZA、2LPT 和 ALPT，这说明神经网络纠偏显著改善了快速模拟对样本方差的恢复。当前协方差估计只使用 240 个模拟实现，样本数量仍偏少，因此矩阵元素会受到较强采样噪声影响；后续需要用更多模拟进一步验证小尺度协方差低估是否为稳定系统偏差。

\section{速度场视觉展示}

速度场对红移空间畸变和运动学SZ效应等探针非常重要，因此本文进一步评估速度场恢复效果。图~\ref{fig:velocity-visual} 展示了一个模拟实现的速度模 \(|v|\) 切片。N 体模拟中速度场包含明显的小尺度高速度结构，这些结构通常与非线性团块和局域塌缩区域相关。ZA 基准速度场主要反映线性大尺度流动，图像更平滑，缺少许多局域高速度峰值。

新流程生成的速度场在整体空间分布上与 N 体结果更相似，并恢复出更多与高密度结构相关的小尺度速度特征。右下角残差图显示，较大的速度残差主要集中在高速度团块附近，说明维里化区域和强非线性塌缩区域仍是速度恢复中最困难的部分。总体而言，速度残差网络能够在 ZA 大尺度速度基准上补充非线性细节，但逐点速度场的恢复难度高于密度场。

\section{速度场两点统计}

最后比较速度场的两点统计。图~\ref{fig:velocity-power} 分别给出速度散度、速度旋度模、速度模以及三个速度分量的功率谱比值。速度散度主要反映大尺度汇聚和发散运动。ZA 在中等波数处出现一定过冲，并在高波数端快速下降；新流程在低、中波数范围内更接近 N 体参考，在高波数端虽然仍低于 1，但整体偏差小于 ZA，说明速度残差网络改善了速度场中与大尺度结构演化相关的可压缩部分。

旋度模功率谱体现非线性演化和维里化运动产生的涡旋成分。由于 ZA 属于线性势流近似，不能产生旋度功率；新流程能够生成非零旋度信号，并在小尺度上恢复出一部分 N 体模拟中的旋度结构。不过图中红线仍明显低于 1，说明当前模型对维里化区域的速度旋度恢复仍不充分。速度模功率谱也表现出类似趋势：新流程相较 ZA 有明显提升，但在全波数范围内仍系统性低估 N 体速度模功率。

三个速度分量的功率谱比值显示，新流程在低波数处基本保持与 N 体参考一致，高波数端逐渐下降；相比 ZA，速度残差网络延缓了小尺度功率损失，在大尺度上也有所改善。综合来看，速度场结果证明本文方法能够改善 ZA 速度基准，但速度场的小尺度随机运动和旋度结构比密度场更难学习，仍是后续模型优化的重点。

\section{运行时间与内存占用}

除统计精度外，快速模拟方法还需要在实际生成任务中保持较低的计算开销。本文在 NVIDIA A100 40GB GPU 上进行模型训练和推理，记录训练阶段的峰值显存和内存占用，并测量从读取初始粒子位置 \(\boldsymbol{q}\) 与一阶位移场 \(\boldsymbol{\Psi}_1(\boldsymbol{x},z=127)\) 到完整输出位移场和速度场的端到端运行时间，结果见表~\ref{tab:runtime-memory}。
